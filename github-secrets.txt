GitHub Secrets And Variables For Single Test ECS Cluster
=======================================================

This project now deploys only one environment: test.
The workflow file is:
.github/workflows/server.yml

Required GitHub Actions secrets
-------------------------------
AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY
ECS_CLUSTER
ECS_API_SERVICE

Required GitHub Actions variable or secret
------------------------------------------
AWS_REGION

Recommended location:
Settings -> Secrets and variables -> Actions -> Variables -> AWS_REGION

Example:
us-east-1

Optional fallback secret names
------------------------------
The workflow also accepts these names if you already created them earlier:

ECS_CLUSTER_TEST
ECS_API_SERVICE_TEST
ECS_SERVICE
ECS_SERVICE_TEST

Recommended simplest setup
--------------------------
Use only these:

AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY
AWS_REGION
ECS_CLUSTER
ECS_API_SERVICE

What each value means
---------------------
AWS_ACCESS_KEY_ID:
IAM access key used by GitHub Actions to push images to ECR and redeploy ECS.

AWS_SECRET_ACCESS_KEY:
IAM secret key for AWS_ACCESS_KEY_ID.

AWS_REGION:
AWS region where ECR and ECS are hosted.

ECS_CLUSTER:
Name of your ECS cluster.

ECS_API_SERVICE:
Name of your ECS service for this FastAPI backend.

Workflow image repository
-------------------------
The workflow pushes to this ECR repository:
mdportal-mw

------------------
test-latest
test-<git-sha>

Minimum likely runtime values
-----------------------------
If using Anthropic:
ANTHROPIC_API_KEY
ANTHROPIC_CHAT_MODEL_ID
ANTHROPIC_MODEL_USE=true
EMBEDDING_PROVIDER=openai or huggingface

If using OpenAI chat:
OPENAI_API_KEY
OPENAI_CHAT_MODEL_ID
ANTHROPIC_MODEL_USE=false
EMBEDDING_PROVIDER=openai
OPENAI_EMBEDDING_MODEL_ID

IAM permissions needed by GitHub Actions user
---------------------------------------------
The AWS key used by GitHub Actions needs permissions for:
- ECR authorization token
- ECR image push to repository mdportal-mw
- ECR list/delete old images in repository mdportal-mw
- ECS update-service on the configured ECS cluster and service
